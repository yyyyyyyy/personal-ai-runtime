## 1. 安装与启动

### 前置要求

- Python >= 3.12
- Node.js >= 20
- npm（随 Node.js 安装）

### 一键安装

```bash
git clone <仓库地址> personal-ai-runtime
cd personal-ai-runtime

# 交互式安装（会引导你配置 API Key）
bash install.sh
```

安装脚本会自动检查环境、引导配置 `.env`、安装依赖、初始化数据库。

### 启动

```bash
make dev
```

启动后访问 `http://localhost:5173`。

### 获取 LLM API Key

| 提供商 | 获取地址 | 费用 |
|--------|---------|------|
| DeepSeek（推荐） | [platform.deepseek.com](https://platform.deepseek.com) | 极低，注册送额度 |
| OpenAI | [platform.openai.com](https://platform.openai.com) | 按量付费 |
| Ollama（本地） | [ollama.com](https://ollama.com) | 免费（需要 GPU） |

### Docker 部署

```bash
docker compose up --build
```

## 2. 对话

对话是核心入口。点击左侧栏「对话」开始。

### AI 能做什么

AI 可以调用多种内置工具：

| 类别 | 示例 |
|------|------|
| 文件操作 | 读写文件、搜索文件、列出目录 |
| 邮件 | 查收件箱、读邮件、发送邮件 |
| 日历 | 查看日程、添加日程 |
| 搜索 | 网页搜索、获取网页内容 |
| Git | 查看提交记录、文件差异 |
| 系统 | 执行命令、读取剪贴板 |

#### 示例

```
你：帮我查一下最近3天的邮件
AI：正在查看收件箱... [显示最近的邮件列表]

你：有什么重要的吗？
AI：[分析邮件内容，给出摘要]
```

### 审批机制

当 AI 需要执行高风险操作（写文件、发邮件、执行命令），会弹出确认框：

- **确认执行**：允许 AI 执行
- **取消**：拒绝执行

低风险操作（如读文件、搜索）自动执行，无需确认。

## 3. 目标管理

### 创建目标

点击「新建目标」，填写：

- 标题
- 描述（可选）

### 管理

- 查看进度和完成百分比

```
你：帮我检查一下「学习 Rust」这个目标的进展

```

## 4. 记忆系统

AI 会自动从对话中提取关于你的信息，保存为记忆。

### 查看记忆

「记忆」页面按类别展示：

- **偏好**：你喜欢用 Rust 写代码
- **习惯**：你每天下午 3 点喝咖啡
- **学习**：你正在学习系统编程

### 为什么要用记忆

记忆让 AI 更了解你。每次对话时，相关记忆自动加载，让 AI 更个性化。

你可以编辑或删除任何记忆。不是黑盒。

## 5. 收件箱

连接 Gmail 后，AI 可以帮你处理邮件。

### 配置

在「设置」→「邮件」中填写 Gmail 地址和 App Password。

获取 App Password：登录 Google 账号 → 开启两步验证 → 搜索「应用专用密码」。

### 使用

在对话中告诉 AI：

```
你：帮我查看收件箱
你：读一下第 3 封邮件
你：帮我回复小王，说我会尽快处理
```

## 9. 数据主权

你的数据完全属于你。

- **导出**：一键将全部数据打包下载为 JSON
- **导入**：从之前导出的文件恢复
- **销毁**：清除所有本地数据

你的对话、记忆、目标——永远可导出，永远可迁移。不被任何厂商锁定。

## 10. 常见问题

**Q: 数据存在哪里？**
`backend/data/` 目录（SQLite + ChromaDB）。

**Q: 如何备份？**
在「设置」→「数据主权」中导出，或直接复制 `backend/data/`。

**Q: 可以不联网使用吗？**
可以。配置 Ollama 本地模型：
```bash
LLM_BASE_URL=http://localhost:11434/v1
LLM_MODEL=qwen2.5:7b
MEMORY_EXTRACTOR=ollama
```

**Q: 换模型需要重新训练记忆吗？**
不需要。改 `LLM_MODEL` 配置，重启。所有记忆保留。

**Q: 如何添加自定义工具？**
见 [开发者指南](DEVELOPER_GUIDE.md)。
