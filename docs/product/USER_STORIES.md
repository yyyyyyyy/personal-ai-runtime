# User Stories

## 小王 — 远程开发者

**32 岁**。全栈开发者。远程工作。每天和 AI 高频互动。

**使用的 AI**：多个模型分别用于日常对话、代码审查、IDE 内编码。

**痛点**：每次换 AI 都要重复讲自己的技术栈偏好。「我喜欢 TypeScript > Python」「我在用 Rust 做 side project」「我的项目部署在 Vercel 上」——这些信息说了几十遍。不同 AI 之间信息完全不通。

**使用本项目后**：
- 一次设置偏好。换三次模型，AI 仍然知道他喜欢 TypeScript。
- 对话中 AI 引用旧对话：「根据你上周提到的部署问题，我建议……」
- AI 对他的理解随时间增长。未来（Phase 1）将支持每周 AI 使用报告和 AI 画像页面。

**使用前**：每次换模型重新开始。**使用后**：换模型不丢记忆。AI 越来越懂他。

## 李老师 — 内容创作者

**28 岁**。独立开发者兼技术博主。同时维护 3 个长期写作项目。

**使用的 AI**：写作辅助、笔记整理等多种用途。

**痛点**：不同的写作项目有不同的风格、受众、参考资料。每次切换项目都需要重新告诉 AI 上下文。AI 经常搞混不同项目的需求。

**使用本项目后**：
- 为不同项目创建不同的对话上下文。AI 自动加载相关记忆。
- 「写作风格」「博客受众」「Github README 的语气」——这些偏好成为持久记忆。
- AI 主动提醒：「你的 Rust 教程系列已经两周没更新了。」

**使用前**：每次切换项目都需要重复上下文。**使用后**：AI 记住每个项目的完整背景。

## 张医生 — 隐私敏感用户

**45 岁**。医生。经常需要 AI 帮助分析医学文献和撰写病案总结。

**痛点**：不能把患者信息上传到云端 AI。云端服务的隐私政策不够清晰。需要 AI 能力但又不能用。

**使用本项目后**：
- 所有数据存储在本机。对话永不出本机。
- LLM 出站仅做审计，不做数据存储。
- 可随时查看 AI 做了什么、哪些数据被发送到了 LLM。
- 完整数据导出，可审计。

**使用前**：因为隐私顾虑不能用 AI。**使用后**：在隐私保障下放心使用 AI。

## David — AI Tinkerer

**35 岁**。早期采用者。喜欢自己搭建、配置、扩展。已经有多个自建的 AI 工具。

**使用的 AI**：自建的本地模型集群 + 多个 API。

**痛点**：自己搭的工具之间没有统一的记忆层。切换工具 = 切换 AI 身份。每个工具对「他」的理解都不同。

**使用本项目后**：
- 所有 AI 工具共享同一个 Memory 层。
- 可以自己写 MCP Server 扩展工具。
- 用 Makefile 和 CI 守护保证系统可靠性。
- 通过 `curl` 导出/导入数据，完全掌控数据流。

**使用前**：碎片化的自建 AI 工具。**使用后**：统一的个人 AI 运行时。

---

*这些故事代表了四类目标用户。产品决策依据这些用户的需求优先级。*
